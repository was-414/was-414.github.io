初次使用spring boot

2021年12月20日

前不久花3小时帮某某入门spring boot，后来自己又看了看，练了练，用spring boot初次做了一个web小小小练习，我想要把这个过程中遇到的我目前容易忘记的给记录下来。

pom依赖

首先在idea中建一个maven项目，不需要选啥模板，建好后在pom.xml文件中导入依赖

  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.4.5</version>
    <relativePath/> <!-- lookup parent from repository -->
  </parent>

  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-test</artifactId>
      <scope>test</scope>
    </dependency>
    <dependency>
      <groupId>org.mybatis.spring.boot</groupId>
      <artifactId>mybatis-spring-boot-starter</artifactId>
      <version>2.2.0</version>
    </dependency>
    <!--Thymeleaf 启动器-->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-thymeleaf</artifactId>
    </dependency>
    <!--导入数据库驱动-->
    <dependency>
      <groupId>mysql</groupId>
      <artifactId>mysql-connector-java</artifactId>
      <scope>runtime</scope>
    </dependency>
  </dependencies>

  <build>
    <!-- 资源过滤,解决mapper中xml文件过滤 -->
    <resources>
      <resource>
        <directory>src/main/java</directory>
        <includes>
          <include>**/*.properties</include>
          <include>**/*.xml</include>
        </includes>
        <filtering>false</filtering>
      </resource>
    </resources>
  </build>

目前大致先用到了这些依赖。

包结构

目前练习中整合了mybatis、Thymeleaf

  1. 先说说resources目录下的结构

    • 在该级目录下我建了2个文件夹，分别为static、templates
    • static目录下主要放静态资源，如css，js，img等等
    • templates目录主要放Thymeleaf视图
    • 在resources目录下还需要创建spring boot的配置文件，我yml还没看过，因此我建的配置文件为：application.properties

  2. application.properties中的配置

    spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/test
spring.datasource.username=root
spring.datasource.password=123456

mybatis.mapper-locations=classpath:com/haichao/mapper/*.xml
mybatis.type-aliases-package=com.haichao.po

    配了一个数据源，配了一下mybatis中的mapper路径和po路径

  3. java目录下的结构

    创建了一个多级目录com.haichao，在该目录下都是常见的分层

    • controller
    • interceptor
    • mapper
    • po
    • service
    • tools
    • 外加一个启动的HelloApplication.java文件

  4. HelloApplication

    //总的启动入口
@SpringBootApplication
public class HelloApplication {
    public static void main(String[] args) {
        SpringApplication.run(HelloApplication.class,args);
    }
}

mapper

记得给mapper接口加注解**@Mapper**

接口中简单的方法直接用注解来实现，困难的方法还是得写xml

写个简单的增删改查示例：

  • @Select("select * from user")
List<User> getAllUser() ;
List<User> getAllUser(RowBounds rowBounds) ;
  • @Insert("insert into user(name,password) value(#{user.name},#{user.password})")
boolean addUser(@Param("user") User user) ;
  • @Delete("delete from user where id = #{id}")
boolean deleteUserById(int id) ;
  • @Update("update user set name=#{user.name},password=#{user.password} where id=#{user.id}")
boolean updateUserById(@Param("user") User user) ;
  • // 模糊查询
@Select("select * from user where name like concat('%', #{name}, '%')")
List<User> getUserByLikeName(String name) ;

controller

写控制器的过程中用到了REST风格

@RequestMapping(value = "/deleteUserById/{id}")
public String deleteUserById(@PathVariable int id,Model model) {
    ......
}

@PathVariable用来接收URL中的请求参数

@RequestMapping(value = "/updateUserById/{id}/{name}/{password}")
public String updateUserById(User user,Model model) {
    ......
}

请求路径上的id、name、password参数会被装入到user中对应属性中

在返回问题上，有时不想返回视图，想转发或重定向到其他控制器的映射路径上，此时需要加重定向和转发的关键字

  • // 转发
return "forward:/user/getAllUser";
  • // 重定向
return "redirect:/user/getAllUser";

Thymeleaf的使用

使用thymeleaf需要加入命名空间

<html lang="en" xmlns:th="http://www.thymeleaf.org">

目前用到了2类表达式

  • 变量表达式 ${}
  • 链接表达式 @{}

想要在链接表达式中使用变量表达式需要使用管道语法，及在链接表达式中的{}中加入||

<a th:href="@{|/user/deleteUserById/${user.id}|}">删除</a>

Thymeleaf 提供了大量的 th 属性，这个不详细记录

我目前用到的th属性有：th:text、th:each、th:value、th:href等等

写一下th:each的案例

<tr th:each="user:${userList}">
    <td th:text="${user.id}"></td>
    <td th:text="${user.name}"></td>
    <td th:text="${user.password}"></td>
</tr>

若想将视图数据赋值给JS变量，应该这样写

<script th:inline="javascript">
    let user = [[${user}]];
</script>

thymeleaf引用静态资源

静态资源是放在static中的，若想在视图中引用静态资源，要先在pom文件中加配置

<resource>
    <directory>src/main/resources</directory>
    <includes>
        <include>**/*.*</include>
    </includes>
    <filtering>true</filtering>
</resource>

这段代码的作用是将resources目录下的所有类型的文件编译到target/classes文件夹中，这样.css和.js后缀的文件就可以正常的访问

现在演示引用bootstrap，先在static目录下建了css和js文件夹

再将bootstrap下载好放入static目录中

现在在视图中引用

<link rel="stylesheet" type="text/css" th:href="@{/css/bootstrap4/bootstrap.css}">
<script th:src="@{/js/jquery/jquery-3.6.0.min.js}"></script>
<script th:src="@{/js/bootstrap4/bootstrap.min.js}"></script>

分页

实现分页先用最原始的方式来做

在tools中建立一个page类，用于分页

public class Page<T> {
    private int total;        //查询总数
    private int pageNo;       //当前页数
    private int count = 5;    //每页个数
    private List<T> results;  //查询结果集

    /*  getter and setter */

}

在service中写实现分页的方法

public Page<User> userList(Integer pageNo) {
    Page<User> userPage = new Page<User>();
    int total = userMapper.getTotal();
    // 边界判断
    if(pageNo == null || pageNo <= 1) {
        pageNo = 1;
    }

    if(pageNo > Math.ceil((total * 1.0) / userPage.getCount())) {
        pageNo = (int) Math.ceil((total * 1.0) / userPage.getCount());
    }

    int start = (pageNo - 1) * userPage.getCount();
    if(start < 0) {
        start = 0;
    }

    RowBounds rowBounds = new RowBounds(start, userPage.getCount());
    List<User> userList = userMapper.getAllUser(rowBounds);

    userPage.setPageNo(pageNo);
    userPage.setTotal(total);
    userPage.setResult(userList);

    return userPage;
}

在视图中直接使用

<a th:href="@{|/user/userList/${page.pageNo - 1}|}">上一页</a>
<a th:href="@{|/user/userList/${page.pageNo + 1}|}">下一页</a>
