快速排序

快速排序的介绍

快速排序是由冒泡排序演变而来的算法，同冒泡排序一样，快速排序也属于交换排序（通过元素之间的比较和交换位置来达到排序的目的），快速排序比冒泡排序要高效很多，这主要是快速排序采用了[分治法]，及快速排序在每一轮挑选一个基准元素，并让其他比它大的元素移动到数列的一边，比它小的元素移动到数列的另一边，从而把数列拆解成了两个部分，这两个部分在下一轮中又同理分别被拆解为两个部分，直至不可在拆解。对比于冒泡排序在每一轮只把一个元素移动到数列的一端来说，效率更高，因此被称作快速排序。

基准元素的选择

最为常见的方式是选择数列的第一个元素。这种选择在大部分情况下都没问题，但是当一个逆序数列要排序成顺序数列，这时整个数列在每一轮中并不会分为两部分，在每一轮中仅仅是确定了基准元素的位置，在这种情况下效率就和冒泡排序差不多。

解决上述问题的办法很简单，及选择基准元素时随机选择一个元素作为基准元素，这样选的话同样有极小的机率选到数列的最大值或最小值，同样会影响到分治的效果。

因此，快速排序的平均时间复杂度是O(nlogn)，最坏情况下的时间复杂度是O(n^2)，所以，快速排序是一种不稳定排序。

元素的移动

对于元素的移动有两种方式。

  • 挖坑法

    import java.util.Arrays;

public class Test {
    public static void main(String[] args) {
        int[] nums=new int[] {4, 7, 6, 5, 3, 2, 8, 1};
        quickSort(nums,0,nums.length-1);
        System.out.println(Arrays.toString(nums));
    }

    public static void quickSort(int[] nums,int startIndex,int endIndex) {
        //递归结束条件的判断
        if(startIndex < endIndex) {
            //选择的第一个元素作为基准元素
            int pivot=nums[startIndex];
            //左右下标
            int left=startIndex;
            int right=endIndex;

            while (left < right) {
                //找出右边比基准元素小的元素
                while (left < right && nums[right] >= pivot)
                    right--;
                //把找出的元素填到左边的坑中，左下标向后移一位
                if(left < right) {
                    nums[left]=nums[right];
                    left++;
                }
                //找出左边比基准元素大的元素
                while (left < right && nums[left] <= pivot) {
                    left++;
                }
                //把找出的元素填到右边的坑中，右下标向前移一位
                if(left < right) {
                    nums[right]=nums[left];
                    right--;
                }
            }
            //上述循环完成后,left=right，将基准元素填入坑中
            nums[left]=pivot;
            //将拆分的两部分递归执行上述操作
            quickSort(nums,startIndex,left-1);
            quickSort(nums,right+1,endIndex);
        }

    }
}

  • 指针交换法

    import java.util.Arrays;

public class Test {
    public static void main(String[] args) {
        int[] nums=new int[] {4, 7, 6, 5, 3, 2, 8, 1};
        quickSort(nums,0,nums.length-1);
        System.out.println(Arrays.toString(nums));
    }

    public static void quickSort(int[] nums,int startIndex,int endIndex) {
        //递归结束条件的判断
        if(startIndex < endIndex) {
            //选择的第一个元素作为基准元素
            int pivot=nums[startIndex];
            //左右下标
            int left=startIndex;
            int right=endIndex;

            while (left < right) {
                //找出右边比基准元素小的元素
                while (left < right && nums[right] >= pivot)
                    right--;
                //找出左边比基准元素大的元素,注意这两个循环的顺序，不能颠倒
                while (left < right && nums[left] <= pivot)
                    left++;
                //将找到的两个元素交换
                if(left < right) {
                    int t;
                    t=nums[left];
                    nums[left]=nums[right];
                    nums[right]=t;
                }
            }
            //上述循环完成后,left=right，将基准元素和两下标重合的元素交换
            //注意此处的startIndex是以基准元素的选择为准，及随着基准元素选择的改变而改变
            /* 一个列子，当选择中间数为基准元素时
                int pivot=nums[(startIndex+endIndex)/2];

                nums[(startIndex+endIndex)/2]=nums[left];
                nums[left]=pivot;
            */
            nums[startIndex]=nums[left];
            nums[left]=pivot;
            //将拆分的两部分递归执行上述操作
            quickSort(nums,startIndex,left-1);
            quickSort(nums,right+1,endIndex);
        }

    }
}

指针交换法和挖坑法相比较，元素的交换次数更少。

总结

在上述内容中我所介绍的快速排序是十分基础的，对于快速排序还有很多改进的版本，我在这里说明一下：在选择基准元素的时候可以采用三数取中法；当排序序列的长度拆分到一定大小后，可以在其中使用插入排序（对于很小和部分有序的序列来说，快排没有插排好）； 对于在代码段的后面有两次递归操作，对于递归操作也可以通过栈来实现非递归操作。

总之，有兴趣的话可以深入的研究研究，对于我来说，目前对于快排的学习到达这个地步就行了。
